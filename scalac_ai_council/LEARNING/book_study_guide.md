# Book Study Guide
## Monthly Book Club dla Rady AI

---

## Format

**Frequency:** Miesięcznie  
**Duration:** 2 godziny  
**Participants:** Cała rada (5 agentów)  
**Rotation:**
- Miesiąc 1: CORE book
- Miesiąc 2: AI book
- Miesiąc 3: Bridge book (łączy CORE + AI)

---

## Process

### 1. Book Selection (Week 1)

Agent wybiera książkę z listy:

**CORE Books:**
- "Crossing the Chasm" - Geoffrey Moore
- "Monetizing Innovation" - Madhavan Ramanujam
- "Building a StoryBrand" - Donald Miller
- "The Challenger Sale" - Matthew Dixon
- "Product-Led Growth" - Wes Bush
- "The Mom Test" - Rob Fitzpatrick
- "From Impossible to Inevitable" - Aaron Ross
- "Predictable Revenue" - Aaron Ross
- "Hacking Growth" - Sean Ellis
- "Demand-Side Sales 101" - Bob Moesta
- "The Qualified Sales Leader" - John McMahon
- "Never Split the Difference" - Chris Voss

**AI Books:**
- Same as CORE (AI to rozszerzenie CORE, nie osobna kategoria)
- Focus na inne zastosowanie konceptów

**Bridge Books:**
- "Zero to One" - Peter Thiel
- "The Hard Thing About Hard Things" - Ben Horowitz
- "The Innovator's Dilemma" - Clayton Christensen
- "Thinking, Fast and Slow" - Daniel Kahneman

### 2. Reading (Weeks 1-3)

Każdy agent czyta książkę z perspektywą:
- Jak to zastosować w Scalac?
- Jak to pasuje do CORE vs AI?
- Jakie są konkretne taktyki?

### 3. Extraction (Week 4)

Agent przygotowuje:
- Key concepts (5-10)
- CORE application
- AI application
- Templates/frameworks
- Action items

### 4. Session (End of Month)

**Agenda:**
1. Presenting agent: 20 min (key concepts + applications)
2. Discussion: 40 min (jak użyć w Scalac?)
3. Application planning: 30 min (konkretne action items)
4. Documentation: 30 min (update system prompts)

### 5. Documentation (After Session)

- Update system prompt agenta
- Add new templates to TOOLKIT
- Update examples
- Share learnings
